Module Irmin.S.View

module View: sig .. end

Database temporary and in-memory views. Similar to View.S but can lazily be created from a path in the database, and later commited back.
type db = t
Database handler.
include Irmin.View.S
val origin_of_actions : t -> origin
Create an origin using the list of actions as message.
val of_path : db -> origin -> key -> t Lwt.t
Read a view from a path in the store. This is a cheap operation, all the real reads operation will be done on-demand when the view is used.
val update_path : db -> origin -> key -> t -> unit Lwt.t
Commit a view to the store. The view *replaces* the current subtree, so if you want to do a merge, you have to do it manually (by creating a new branch, or rebasing before commiting). origin helps keeping track of provenance.
val rebase_path : db -> origin -> key -> t -> unit Irmin.Merge.result Lwt.t
Rebase the view to the tip of the store.
val merge_path : db -> origin -> key -> t -> unit Irmin.Merge.result Lwt.t
Same as update_path but *merges* with the current subtree.
