Module Irmin.Private

module Private: sig .. end
Private defines functions only useful for creating new backends. If you are just using the library (and not developing a new backend), you should not use this module.

API to create new Irmin backends.

There are two ways to create a concrete Irmin.S implementation.

Irmin.Private.Make creates a store where all the objects are stored in the same store, using the same internal keys format and a custom binary format based on bin_prot, with no native synchronization primitives: it is usually what is needed to quickly create a new backend.

Irmin.Make_ext creates a store with a deep embedding of each of the internal stores into separate store, with a total control over the binary format and using the native synchronization protocols when available. This is mainly used by the Git backend, but could be used for other similar backends as well in the future.
