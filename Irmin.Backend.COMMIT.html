Module type Irmin.Backend.COMMIT

module type COMMIT = sig .. end
Commit values represent the store history.

Every commit contains a list of predecessor commits, and the collection of commits form an acyclic directed graph.

Every commit also can contain an optional key, poiting to a node value. See the NODE signature for more details on node values.

include Irmin.AO
module Key: Irmin.Hash.S  with type t = key
Key provides base functions for commit keys.
